# 채점 결과 오류코드 정리 표

> 목적: `/try` 채점(및 코치·추출)에서 사용자가 보는 **오류 코드 → 원인 → 대응**을 한 표로.
> QA·CS·게이트키퍼 보고용 참조 문서. (2026-07-09, prod 오픈 QA 중 작성)
>
> 코드 SoT: [app/lib/grading.ts](../app/lib/grading.ts) `ErrorCode`·`ERROR_HTTP`·`ERROR_MESSAGE` ·
> 재시도 셋: [app/hooks/useScoreForm.ts](../app/hooks/useScoreForm.ts) `RETRYABLE`.
> 이 문서는 **요약본** — 코드가 바뀌면 표도 갱신할 것(계약 변경은 12_api_contract 참조).

## 사용자에게 보이는 것

채점 실패 시 결과 영역에 **"채점을 마치지 못했어요" + 메시지 + `오류 코드: <코드>`** 가 뜬다.
재시도 가능한 코드면 **[다시 시도하기]** 버튼이 함께 노출된다.

## 오류코드 표

| 코드 | HTTP | 사용자 메시지 | 원인(발생 지점) | 재시도 버튼 | QA/CS 대응 |
|---|---|---|---|---|---|
| `E1` | 400 | 과제 정보를 모두 올바르게 입력해 주세요. | 요청 검증 실패 — 학년/과목/장르 미선택, 과제문 길이 위반 등 (`/api/score` validateRequest) | ✕ | 입력 폼 확인. FE 가드 통과 후에도 나오면 버그(FE·서버 검증 불일치) |
| `E2` | 422 | 본문을 50자 이상 입력해 주세요. | 본문 최소 길이 미달 | ✕ | 본문 50자 이상 입력 |
| `E3` | 400 | 2,000자까지 첨삭할 수 있어요. | 본문 최대 길이 초과 | ✕ | 본문 2,000자 이하로 |
| `E10` | 400 | 목표 글자 수는 50~2,000자로 입력해 주세요. | 목표 분량 범위 위반 | ✕ | 목표 글자 수 수정(또는 비움) |
| `E11` | 422 | 글에 본문이 충분히 들어 있지 않아요. 학생이 쓴 글 전체를 붙여넣어 주세요. | 사진/텍스트 추출 결과에 본문이 부족 (추출 흐름) | ✕ | 원문 전체를 직접 붙여넣기 |
| `E-AUTH` | 401 | 로그인이 필요해요. 다시 로그인해 주세요. | 세션 만료/쿠키 부재 (`verifyWritingAccess`) — FE는 에러 패널 대신 **인증 만료 처리**(onAuthExpired)로 흡수하는 게 기본 경로 | ✕ | 재로그인. 로그인 직후에도 반복되면 env/쿠키 격리(ops-prod-deploy-checklist §5) |
| `E-PARSE` | 400 | 결과를 다시 만들어야 해요. 다시 시도해 주세요. | 요청 body가 JSON이 아님(전송 깨짐) | ✔ | 재시도. 반복 시 네트워크/프록시 확인 |
| `E4` | 504 | 지금 첨삭이 지연되고 있어요. 다시 시도해 주세요. | 모델 호출 타임아웃(서버) 또는 FE fetch 65초 타임아웃 | ✔ | 재시도. 지속 시 Anthropic 상태 확인 |
| `E5` | 502 | 결과를 다시 만들어야 해요. 다시 시도해 주세요. | 모델 응답이 스키마 불일치 — 재호출까지 무효(깨진 결과는 화면에 내보내지 않음) | ✔ | 재시도. 빈발 시 프롬프트/스키마 회귀 의심 → Sentry `errorCode:E5` |
| `E6` | 502 | 결과를 다시 만들어야 해요. 다시 시도해 주세요. | 코치(`/api/coach`) 응답 무효 — 채점의 E5에 대응하는 코치 쪽 코드 | ✔ | 재시도. Sentry `errorCode:E6` |
| `E8` | 503 | 일시적 오류예요. 잠시 후 다시 시도해 주세요. | 업스트림 Anthropic 429/5xx·네트워크, **또는 `ANTHROPIC_API_KEY` 미설정**(로컬에서 흔함 — 2026-07-09 QA 실사례) | ✔ | 서버 로그에 `model call E8: <detail>` — "미설정"이면 env, 아니면 업스트림 장애 대기 |
| `E-CAP` | 429 | 오늘 사용량이 많아요. 잠시 후 다시 시도해 주세요. | 사용량 상한(레이트 리밋) — 추출 클라이언트에서 사용, 채점은 예약 | ✔ | 잠시 후 재시도 |
| `E-NET` | (없음) | 인터넷 연결을 확인하고 다시 시도해 주세요. | **서버 응답 자체가 없음** — FE fetch reject(오프라인·DNS·CORS). 서버 코드 아님(FE 전용) | ✔ | 사용자 네트워크 확인. 특정 origin에서만 나면 CORS 격리 |

## 격리 순서 (QA 빠른 판별)

1. **코드가 4xx 검증군(E1·E2·E3·E10·E11)** → 입력 문제. 폼 가드가 막지 못했다면 FE 버그로 기록.
2. **`E-AUTH`** → 세션. 재로그인으로 해소되는지 → 안 되면 env(NEXT_PUBLIC_API_URL)·쿠키(도메인) 순.
3. **`E8`(503)** → 서버 로그 `model call E8:` detail 확인 — `ANTHROPIC_API_KEY 미설정`(env)인지 `upstream HTTP 429/5xx`(Anthropic 장애)인지 즉시 구분됨.
4. **`E4`/`E5`/`E6`** → 일시 재시도로 대부분 해소. 빈발 시 Sentry 태그(`route`, `errorCode`)로 집계 확인.
5. **`E-NET`** → 서버까지 도달 못 한 것. 사용자 네트워크/사내 프록시/CORS.
