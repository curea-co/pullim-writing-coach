## 0. TL;DR (한 장 요약)

- **두 데모 모두 기획·교육적 책임성은 상위권이지만, 공통의 치명적 공백이 있다: "AI가 실제로 이만큼 잘한다"를 증명하지 못한다.** (Writing Coach는 핵심 기능이 비번 잠금 + 샘플이 하드코딩, Admissions Coach는 전면 mock.)
- **Writing Coach `/try` 입력 화면의 문제**: 옛날식 폼(입력 박스 과다) + "복붙 강요"가 현실 학생 동선과 안 맞는다.
- **재설계 기조 = "입력 폼"을 버리고 "전달 + AI가 정리"로.** 학생은 가진 형태(사진·파일·링크·복붙) 그대로 던지고, AI가 본문 추출 + 메타데이터 추론 + 못 맞춘 1~2개만 되묻는다. 결정의 수를 6개 → 1개로.
- **핵심 아이디어(척추) = "두 문서 모델".** 입력의 주인공을 '학생 글'이 아니라 **'선생님이 준 수행평가 안내서'**로 끌어올린다. 안내서 한 장에서 과제문·장르·분량·조건·**선생님의 실제 루브릭**을 뽑아낸다 → "선생님 기준으로 채점"이 가능해져 신뢰도가 급상승.
- **기능 분리 = '준비(쓰게 돕기)' 모드 / '평가(채점)' 모드.** 분리하되 안내서를 공통 뿌리로 연결, 보완 루프로 순환.
- **확장 과제(사각지대) 7개** + **리스크 레지스터**(대필·공정성·타당성)를 본문에 정리.

## 1. 배경 — 평가한 두 서비스

### 1-1. Pullim Writing Coach (수행평가 글쓰기 첨삭)

- 5영역 루브릭(과제 이해·내용 충실도·구조·논리·표현·문장·성장 가능성, 각 20점, 총 100점) + 코칭 톤 피드백(잘한 점/고칠 점/수정 가이드).
- 점수대별 샘플 5종(40·61·74·85·86), 형광펜 ↔ 피드백 연결, "왜 이 점수예요?" 루브릭 구간 토글, PDF/스크린샷/복사 내보내기.
- 스택: Next.js(App Router·Turbopack) + Vercel. 접근성(시맨틱·ARIA) 양호.

**진단 점수표 (10점 만점)**

| 항목 | 점수 | 한 줄 |
|---|---|---|
| AI 활용(증명) | 3 | 샘플은 하드코딩(시뮬레이션), 실시간 채점은 비번 잠금 → 검증 불가 |
| 피드백 품질(샘플 기준) | 8 | 구체·텍스트 기반·우선순위까지, 가장 강한 부분 |
| 고객 편의성 | 5 | 매끄럽지만 핵심 기능 잠금 + 학생/교사 대상 혼란 |
| 레이아웃 완성도 | 7 | 위계·접근성 좋음, 내부 메타 라벨 노출이 흠 |
| 기능 독창성 | 5 | 카테고리는 흔함, 투명성·코칭 톤만 차별 |
| 레이아웃 독창성 | 3 | 전형적 대시보드 템플릿 |
| 교육적 책임성 | 9 | 면책·포지셔닝·출처 처리 성숙 |
| 개발 완성도 | 6 | 스택·접근성 OK, 내부 산출물(스펙 버전·파일명) 노출 감점 |

> **가장 냉정한 지적**: 푸터에 "AI 결과는 EPO 자체 채점값을 그대로 사용한 시뮬레이션"이라 명시되어 있어, 공개 데모만으로는 **AI 채점의 진짜 실력을 확인할 수 없다.** `§2 원칙 6-(b)`, `rubric v0.5`, `Week 1 산출물` 등 **내부 스펙 용어가 사용자 화면에 노출**되는 것도 정리 필요.

### 1-2. Pullim Admissions Coach (학종 진학 코치) — 병행 평가

- 학생부 종합 전형 면접 준비 팩 + 생기부 진단(학업/진로/공동체역량 + 인성) + 부족 활동 보완안. 4단계 플로우(입력→동의→처리→결과), 개인정보 마스킹 체크리스트, **학생/학부모 권한 분리**(학부모는 진행 요약만, 본문 미노출).
- 상태: 전면 **mock**("Phase A 시각 프로토타입"), 출시 예정 2026-08-01.

**진단 요약**: AI 활용은 0(목업)이나 **정직하게 명시**. 시각/브랜드는 Writing Coach보다 우위. **대필 거부 + 권한 분리** 등 윤리·컴플라이언스 설계가 동종 최고 수준.

### 1-3. 두 서비스 공통 결론

> 같은 DNA — *"AI는 방향·근거만, 최종 판단은 사람"* + 면책·정직성. 기획·교육 감수성은 상위권.
> **그러나 공통의 공백도 같다: 둘 다 "AI가 이렇게 해줄 것"을 보여줄 뿐 "AI가 실제로 이만큼 잘한다"를 증명하지 못한다.** → 두 제품 모두에 가장 시급한 것은 **"실제 모델을 그 자리에서 돌린 진짜 출력 + 교사 채점 대비 일치율 수치"**.

## 2. 문제 정의 — 현재 `/try` 입력 화면의 한계

1. **순서가 거꾸로다.** 학생에게 먼저 학년·과목·장르·과제문·목표분량(약 6개 결정)을 묻고 *그다음* 큰 텍스트 박스에 복붙시킨다 → 가치 전에 부담부터.
2. **복붙은 비현실적 가정이다.** "글이 이미 디지털이고 선택 가능한 텍스트"를 전제하지만, 실제 수행평가 글은 **공책 손글씨·HWP 파일·구글독스·종이 답안지·카톡 메모**에 흩어져 있다.
3. **폼이 죽어 있다.** 가이드·점진 노출·어시스턴트 개입이 없는 옛날식 입력 폼.
4. **비번 게이트.** 데모 한정이지만, 평가자가 핵심 행동에 닿지 못한다(별도 이슈).

## 3. 재설계 방향 (설계 기조)

### 3-1. 핵심 리프레임 — "전달 + 정리"

> **"붙여넣지 말고 던져라. 정리는 코치(AI)가 한다."**
> 학생은 *가진 형태 그대로* 던지고, AI가 본문을 추출 + 메타데이터를 추론 + 못 맞춘 1~2개만 되묻는다. **결정의 수를 6개 → 1개로.**

### 3-2. 척추 — "두 문서 모델": 안내서(기준) + 결과물(글)

학생의 실제 동선: *선생님이 수행평가 안내서/채점기준표 배부 → 학생이 보고 글 작성 → 제출.*
→ **안내서를 첫 입력으로 받는다.** 안내서 한 장(사진/파일/링크)에서 AI가 추출:

- 과제문(주제) · 장르 · **분량 요건** · 제출 형식·기한
- **유의 조건**(근거 2개 이상, 개요 포함, 출처 표기 등)
- **선생님의 실제 채점 기준표/배점** ← 결정적

**왜 게임체인저인가**: 지금은 풀림 자체의 '공통 5영역'으로만 채점한다. 안내서를 읽으면 **그 반의 진짜 선생님 기준으로** 채점할 수 있다 → *"선생님 기준 ○○항목에 비춰보면…"*. 1차 평가의 약점("교사가 루브릭을 못 바꾼다")을 정면 해결하고, **교사·학부모 신뢰도를 급상승**시킨다. 또한 교사 과제코드 연동의 토대가 된다.

→ 입력 1순위는 복붙도 사진도 아니라 **"과제 안내서 던지기"**. 결과물(학생 글)은 그다음.

### 3-3. 기능 분리 — "준비 모드" / "평가 모드" (분리하되 연결)

| | 🟦 준비 모드(작성 지원) | 🟥 평가 모드(채점) |
|---|---|---|
| 언제 | 쓰기 전·쓰는 중 | 다 쓴 뒤(제출 전) |
| 학생 상태 | "어떻게 시작하지" 막막 | "점수 나오겠지" 긴장 |
| AI 행동 | **질문형 코칭**(개요·조건 점검) | 루브릭 채점 + 보완안 |
| **점수** | **안 줌**(개요에 점수는 무의미·의욕만 꺾음) | 5영역 + 안내서 기준 점수 |
| 성격 | 형성적(배우는 과정) | 총괄적(점검) |

**원칙: 분리하되 한 여정으로 잇는다.** 안내서가 둘의 공통 뿌리라 같은 기준이 양쪽에 흐른다.

### 3-4. 정보구조 / 화면 흐름

```
[과제 안내서 던지기] → AI가 "과제 카드" 생성(과제문·장르·분량·조건·루브릭)
        │  (또는 교사 과제코드로 자동 바인딩)
        ▼  확인/수정
   ┌──────────────┴──────────────┐
   ▼                              ▼
🟦 아직 안 썼어요                🟥 다 썼어요
   준비 모드                      평가 모드
 · 과제 해석(쉬운 말)            · 결과물 던지기(사진/파일/복붙)
 · 개요 잡기                     · 안내서 기준 + 5영역 채점
 · 조건 체크리스트 ✅❌           · 보완안
 · 초안 한 문단씩 피드백            │
        │                          ▼
        └──── (초안 완성) ──→ 평가 모드 ──→ 보완 ──→ 수정 ──→ 재채점(점수 변화) ↺
```

### 3-5. 입력 패러다임 — 3안 비교 및 추천

| 패러다임 | 강점 | 약점 |
|---|---|---|
| A. 대화형(챗) | 체감 노력 최저, 친근, 모바일 강함 | "제출→결과" 작업엔 챗이 느리고 불확실하게 느껴질 수 있음 |
| **B. 단일 스마트 캔버스 + AI 코파일럿 ⭐ 추천** | "제출하고 결과 받는" 모델 유지 + 폼 제거, 통제감, 양 기기 대응 | OCR·파싱·신뢰도 UI 설계 필요(엔지니어링↑) |
| C. 점진 캡처 위저드 | 첫 사용자에게 가장 명확 | 단계가 남음, 파워유저(교사 반복)엔 답답 |

> **추천: B를 척추로, A의 어시스턴트 말투를 입히고, C의 점진 노출을 '처음 사용자'에만.** = "한 화면이지만 어시스턴트처럼 행동하는 AI 캔버스".

### 3-6. Universal Capture (단일 캡처 존) 상세

폼이 아니라 **무엇이든 받는 하나의 큰 영역**:

- 📷 **사진/스캔**(손글씨 공책·종이 답안 → 비전 OCR) ← *현실의 기본값, 가장 크게 노출*
- 📄 **파일**(HWP·DOCX·PDF·이미지 드래그앤드롭)
- 📋 **붙여넣기**(기존 방식도 유지하되 강요 아님, 스샷 클립보드 붙여넣기 허용)
- 🔗 **링크**(구글독스·노션 공개 링크 파싱) — Phase 2
- 🎤 **말로 불러주기**(받아쓰기 — 메모·본문 보강용)
- ⌨️ 그냥 여기에 **바로 타이핑**

핵심: **복붙을 '유일한 길'에서 '여러 길 중 하나'로 강등**, **카메라/OCR 승격**. **여러 장 사진 병합**(공책 2~3장 연속 촬영 → 한 글로 자동 합침) 지원.

### 3-7. 정리 단계 — AI가 본문 + 메타 동시 추출

- "코치가 읽는 중…" **스트리밍** 상태로 인식 텍스트가 실시간 채워짐(멈춘 스피너 X).
- 본문은 **편집 가능한 미리보기**, OCR이 헷갈린 단어는 **형광 표시 + 탭 한 번 수정**.
- 메타데이터는 **편집 가능한 칩(pill)** 으로, 신뢰도 표시: 확정=실선 / 추정=점선+"확인" 뱃지. 탭하면 드롭다운 수정.
- 과제문이 없으면 **딱 그것 하나만** 인라인 질문("이 글은 어떤 과제였어요? 없으면 건너뛰기").

> **빈칸을 채우게 하지 말고, 제안을 확인하게 한다.** 이것이 어시스턴트 기조의 핵심.

### 3-8. AI 어시스턴트 원칙 (기조 6선)

1. 항상 주요 행동은 **하나만** 보인다(필드의 벽 금지).
2. **묻지 말고 추론한다** — 전부 기본값, 못 맞춘 것만 한 번에 하나씩 인라인.
3. **신뢰도를 드러낸다** — 추정값은 점선·"확인" 뱃지.
4. **모든 걸 스트리밍한다** — OCR·파싱·채점 → 체감 속도 + "살아있는 코치".
5. **관대하게** — OCR 틀려도 인라인 수정, 파일 깨지면 "사진으로 다시 줄래요?" 대안.
6. **이어서 하기** — 임시저장 + "이어서 하기" 카드 노출.

### 3-9. 모바일/데스크톱 (둘 다 1급, 같은 컴포넌트 반응형)

- **모바일**: 캡처 존 풀폭·카메라 우선 → 메타 칩 가로 스크롤 한 줄 → 결과 아래로 스트리밍. CTA 엄지 영역 고정.
- **데스크톱**: 좌측 본문 에디터 / 우측 코치 패널(메타 칩 + 코치 질문 + CTA), 드래그앤드롭 강조.

## 4. 확장 과제 — 사각지대 7개 (직원 발전 포인트)

> 입력 화면을 넘어 제품 전체를 "학생이 스스로 자기 글을 볼 줄 알게 만드는 학습 루프"로 키울 때 챙길 지점.

### 4-1. 학생은 '폼 채우는 사람'이 아니라 '불안한 10대'다 (감정 설계)
- **점수를 먼저 던지지 마라** — 강점·"다음 한 걸음"을 먼저, 숫자는 뒤로. *절대 결핍으로 화면을 끝내지 않는다*.
- **자기평가 먼저, AI는 나중에** — "네가 먼저 5영역에 점수를 매겨봐 → AI와 비교". 학습 효과를 크게 키우고 루브릭을 내면화시킴.
- **의존 경계** — 매번 떠먹여 주면 자기 점검력이 안 큰다. 목표는 "AI 없이도 자기 글을 볼 줄 알게".

### 4-2. '교사도 위한다'면서 교사 화면이 0이다
- 교사가 **안내서 1번 업로드 → 과제코드 발급 → 반 단위 패턴**("우리 반 70%가 '근거 부족' 실점") → 무엇을 다시 가르칠지 알려줌.
- ⚠️ **선**: 한국에서 **AI로 내신 점수를 산출하면 안 된다**(공정성·이의신청·성적 규정). 교사 기능은 "1차 검토·피드백 초안"이지 "최종 점수"가 아니어야 함. 세특 문구 자동작성은 *생기부 기재=교사 영역* 선을 넘으면 위험.

### 4-3. 대필 거부는 '윤리'가 아니라 '시장 생존'이다
- 너무 잘 코칭해 **수행평가 변별력을 무너뜨리면 교사가 막는다.** 살아남는 길은 *학습 도구*로 보이는 것.
- 준비 모드는 문장을 대신 써주지 말고 **소크라테스식 질문**("네 주장이 뭐야? 근거 두 개 떠올려봐")으로 학생이 쓰게.
- **사교육 격차**: 무료 티어/학교 보급 모델을 처음부터 염두 — 접근성이 곧 정당성.

### 4-4. '타당성'을 증명 못 하면 학교는 안 쓴다 (측정·환각)
- **교사 대비 일치율 수치** 공개(샘플 N편 실제 교사 채점 → ±X점, 일치율 Y%).
- **모든 피드백에 텍스트 근거(span) 의무화**(형광펜 연결을 전체로 강제) → 환각(없는 걸 칭찬·지적) 차단.
- **반론 허용**: "이 부분은 일부러 그런 거예요" → AI 재고·표시.
- **확신도 표시**: 점수를 점이 아니라 **구간(중상위권)** 으로 줄 수도 → 교사 채점과의 격차 충격 흡수.

### 4-5. 수행평가의 상당수는 '글'이 아니다 (스코프 정직)
- 발표(PPT)·포스터·UCC·토론·**과학 실험보고서(표·그래프·수식)**·모둠 과제는 텍스트 코치가 못 다룬다. **영어 수행평가(영작문)**는 별개 루브릭.
- "우리는 **글쓰기** 수행평가 코치"라고 경계를 분명히 → 오히려 신뢰.

### 4-6. 한 번 쓰고 끝? 시간축에 진짜 가치가 있다 (리텐션 + 해자)
- **학기 단위 성장 포트폴리오**("3월 대비 표현 영역 +12") — 재방문 이유 + 자기 정리 자료.
- **안내서 코퍼스 = 경쟁자가 못 가지는 해자**(학년·과목·장르·지역별 실제 루브릭). 단 **익명화·옵트인 철저**.

### 4-7. 첫 사용은 '마감 전날 밤, 30초'다 (현실 제약·실패 모드)
- 첫 가치까지 **30초** — 안내서 ingestion도 **건너뛰기 가능**(장르 기본 루브릭 폴백).
- 학교 와이파이·구형 폰·OCR 실패·모델 다운 대비 graceful degradation + 오프라인 임시저장.
- 비전+채점은 **비용↑** → 다운스케일·레이트리밋·남용 방지.

## 5. 기술 고려사항 (개발자용)

- **손글씨 OCR이 진짜 관문.** 범용 OCR은 인쇄체엔 강하나 **한국어 손글씨**엔 약함 → **멀티모달 LLM에 이미지 직접 입력**(예: Claude vision)으로 *OCR + 메타 추론을 한 번의 호출*로. 지연은 **이미지 다운스케일 + 스트리밍**으로 관리.
- **파일 파싱**: PDF/DOCX는 서버 텍스트 추출. **HWP는 까다로움** → 1차엔 "HWP는 PDF로 내보내 주세요" 폴백 + 로드맵에 HWP 파서.
- **메타 추출**: 본문(+과제문)을 **구조화 출력(JSON 스키마)**으로 한 번에 추론 → 칩 채움.
- **안내서 → 루브릭 변환**: 표(2D 채점기준표) 인식 필요. 비전 LLM이 표를 구조화. 불완전·오업로드 대비 "안내서에서 이런 기준을 찾았어요, 맞아요?" 확인 UI.
- **개인정보**: 업로드 즉시 클라이언트에서 **다운스케일 + EXIF 제거**, 채점 후 미저장(기존 정책 일관). 손글씨엔 얼굴/이름이 찍힐 수 있으니 사전 마스킹 안내.
- **스트리밍 UX**: 토큰 스트리밍으로 OCR/채점 진행 노출, 낙관적 전환으로 체감 지연 제거. 결과는 `aria-live`.
- **접근성**: 캡처 존 키보드 접근, 칩 편집 ARIA.
- **덜어낼 것(YAGNI)**: 별도 '목표 분량' 숫자 입력 → 추정 칩으로. 장르·과목 라디오 더미 → 자동 칩 + 드롭다운.

## 6. 우선순위 로드맵 (제안)

> 'B 캔버스 + 두 문서 모델'을 점진 출시하는 제안. 각 Phase는 독립 출시 가능.

| Phase | 범위 | 산출 가치 | 비고 |
|---|---|---|---|
| **P0 — 신뢰 기반** | 내부 스펙 용어 노출 제거, 모든 피드백에 텍스트 근거 강제, 면책·확신도 표기 정리 | 가장 싸게 신뢰도↑ | 기존 코드 정리 위주 |
| **P1 — Universal Capture** | 단일 캡처 존(사진·파일·복붙·타이핑), 비전 OCR + 본문 미리보기·인라인 수정, 메타 추정 칩 | "복붙 강요" 제거, 결정 6→1 | 멀티모달 LLM 1콜 설계 |
| **P2 — 두 문서 모델** | 안내서 던지기 → 과제 카드(과제문·장르·분량·조건·루브릭) 생성, **조건 체크리스트 ✅❌** | "선생님 기준 채점" + 실점 방지 | 표 파싱·확인 UI |
| **P3 — 준비/평가 분리** | 준비 모드(질문형 코칭·개요·조건 점검, 점수 없음) ↔ 평가 모드 연결, 보완→재채점 루프(점수 변화) | 학습 루프 완성, 리텐션 | 대필 가드레일 필수 |
| **P4 — 교사 + 시간축** | 교사 과제코드·반 단위 패턴, 학기 성장 포트폴리오, (옵트인) 안내서 코퍼스 | B2B·해자·재방문 | 내신 점수 산출 금지선 준수 |
| **상시 — 타당성** | 교사 대비 일치율 측정·공개, 장르/학년 캘리브레이션 | 학교 채택의 전제 | 데이터·검증 트랙 |

## 7. 리스크 레지스터

| 리스크 | 성격 | 완화 |
|---|---|---|
| **대필·AI 작성 수행평가** | 윤리 + 시장 생존 | 준비 모드는 질문형 코칭만, 완성 문장 미제공. "AI는 방향만" 기조 일관 |
| **내신 점수 AI 산출** | 법·규정 | 교사 기능은 1차 검토·피드백 초안에 한정, 최종 점수 산출 불가 명시 |
| **사교육 격차·공정성** | 사회·평판 | 무료 티어/학교 보급, 접근성 우선 |
| **AI 채점 환각·낮은 타당성** | 신뢰·채택 | 근거 span 의무화, 교사 일치율 공개, 확신도 구간 표기 |
| **개인정보(손글씨 이미지)** | 법·신뢰 | 즉시 다운스케일·EXIF 제거, 미저장, 마스킹 안내 |
| **변별력 훼손 → 교사 거부** | 시장 | '학습 도구' 포지셔닝, 정답 대본 미제공 |

## 8. 열린 질문 (의사결정 필요)

1. **준비/평가**를 한 여정의 두 단계로 묶을지, 두 진입점으로 분리할지(권장: 한 여정).
2. **교사용 제품**을 이번 사이클에 포함할지(B2B 전략과 연동).
3. **무료 티어/학교 보급** 모델의 사업적 결정(공정성 대응과 직결).
4. **HWP 1차 지원 범위**(폴백 vs 정식 파서).
5. **타당성 검증**을 위한 교사 채점 데이터 확보 경로.
6. 안내서 **코퍼스 활용**의 동의·익명화 정책 범위.

## 부록 B — 한 줄 정의 (제품 북극성)

> **"채점 도구"가 아니라 "학생이 스스로 자기 글을 볼 줄 알게 만드는 학습 루프".**
> 안내서(선생님 기준)를 뿌리로, 준비(쓰게 돕기)와 평가(근거 있는 채점)가 갈라졌다 보완 루프로 다시 만나고, *자기평가 먼저·대필 거부·근거 의무화·성장 추적*이 신뢰를 쌓으며, *글이 아닌 수행평가·내신 점수 산출은 안 한다*고 정직하게 선을 긋는다.
