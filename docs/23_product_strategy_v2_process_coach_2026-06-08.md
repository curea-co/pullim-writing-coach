## 1. 전략 앵커 — 왜 지금, 왜 글쓰기 코치인가

**진짜 수요(빈칸이 아니라 흐름):**
- 2022 개정 교육과정 + 2028 내신 논·서술형 확대 → 변별 무대가 객관식 지필 → **수행평가·서술형 글쓰기**로 이동([교육부 2028 확정안](https://www.moe.go.kr/boardCnts/viewRenew.do?boardID=294&boardSeq=97551)). ※ 수능엔 서술형 미채택 — 순풍은 **내신·수행평가**.
- 2025 2학기부터 수행평가 **수업시간 내 시행** 원칙 → 학원·부모 대리 개입 차단 → **"학생 본인이 더 잘 쓰게 돕는"** 도구의 자리가 열림.
- 학생 96.5%가 이미 생성형 AI를 쓰지만([세계일보](https://www.segye.com/newsView/20251026504283)) — 대부분 *대신 써주는* 용도라 **0점 불안 + "어떻게 고쳐야 좋은지 모름"** 이중 페인. 우리는 그 반대편(끌어내기·고쳐쓰기 코칭).

**한 줄 명제:** *"글쓰기가 학습의 중심이 되는 시대에, 학생을 실제로 가장 잘 쓰게 만들고 — 교사가 인정하고 학생이 자유롭게 쓰는 — 글쓰기 과정 코치."*

## 2. 제품 척추 — 평가기 → 과정 코치

| | 평가기 (현행·KEEwi·대다수) | **과정 코치 (전환 후·Pullim)** |
|---|---|---|
| 입력 | 다 쓴 글 | 과제·루브릭 + 쓰는 중인 글 |
| 시점 | 사후 1회 | 쓰는 동안 + 고쳐쓰기 반복 |
| 산출 | 점수·등급 | 사고 발판·콕집은 피드백·성장 |
| 학생 경험 | 심판받음 | 함께 써내며 성장함 |
| 교사 관점 | (불신) 베끼는 도구? | 가르치는 도구 — 인정 |
| 차별 | 첨삭 깊이 경쟁 | **성장 + 신뢰 설계** |

**원칙(전 단계 공통):** *생각·구조·고쳐쓰기는 끌어내되, 문장은 대신 쓰지 않는다.* 이 한 줄이 (a)실제 실력 향상 (b)교사 인정 (c)자유 사용을 **동시에** 성립시키는 설계 축.

## 3. 핵심 기능 — 5단계 코칭 루프 (상세)

> 전 PM이 가장 약했던 부분. 각 단계는 "무엇/어떻게 동작/대신 안 씀 원칙/차별점"으로 명세.

### ① 과제·루브릭 해독 (Decode)
- **무엇:** 학생이 *과제 안내문 + 채점기준표(루브릭)*를 사진/붙여넣기 → 코치가 "진짜 요구사항 + 점수 갈림목 + 분량·장르 조건"을 카드로 추출.
- **동작:** 과제 텍스트 파싱 → 요구 조건 체크리스트 + 루브릭 항목별 "이 글이 노려야 할 것" 매핑. 교사 루브릭이 있으면 *그것*에 정렬(교사 통제권 = 인정의 시작).
- **대신 안 씀:** 답을 주는 게 아니라 *과제를 정확히 이해*하게만.
- **차별점:** 대부분 도구가 실제 과제·루브릭을 무시하고 일반 채점만 함. 수행평가 특화 진입점.

### ② 생각 끌어내기 (Ideate, 소크라테스식) — **킥의 핵심**
- **무엇:** 코치가 *질문*으로 학생 본인의 주장·근거·경험·반론을 끌어냄.
- **동작:** 과제 유형별 질문 시퀀스(설명문=대상 분해/예시, 논설문=주장·근거·반론·재반박, 감상문=장면·정서·연결). 학생 답을 모아 **학생 언어로 된 개요**로 조립.
- **대신 안 씀:** 내용·아이디어를 *제공하지 않음.* "이 주장의 근거 하나만 네 경험에서 떠올려볼까?" (생성 X)
- **차별점:** ChatGPT는 아이디어를 *준다*(베끼기). 우리는 *끌어낸다*(실력). 교사가 "이건 학생 생각"이라고 인정하는 지점.

### ③ 초안 함께쓰기 (Draft with scaffolds)
- **무엇:** 학생이 문단별로 직접 작성, 코치는 여백에서 발판.
- **동작:** 문단 단위 신호("이 문단 근거가 비었어 / 주제문이 흐릿해 / 앞 문단과 연결이 끊겼어") + *학생이 직접 메우도록* 유도. 문장 자동완성/대필 없음.
- **대신 안 씀:** 문장을 써주지 않고 *무엇이 빠졌는지*만 짚음.
- **차별점:** "빈 화면 공포" 해소 + 베끼기 없는 진행 = 리텐션 + 신뢰.

### ④ 고쳐쓰기 루프 (Revise loop) — **"가장 잘 보조"의 엔진**
- **무엇:** 쓰고 → 루브릭에 박힌 콕집은 피드백 → 학생이 다시 씀 → 재점검 → 반복.
- **동작:** 5영역(과제이해·내용충실·구조논리·표현문장·성장가능성) 중 *가장 점수를 끌어올릴 1~3개*만 우선 제시(과부하 방지). 수정 전/후를 나란히 보여 **"한 번의 수정으로 오른 폭"**을 즉시 체감.
- **대신 안 씀:** "이렇게 고쳐라"가 아니라 "여기서 뭐가 약한지 + 어떤 방향으로 네가 고쳐볼지."
- **차별점:** 강사 첨삭은 4일 지연·1회성. 우리는 즉시·반복. *성장이 눈에 보임.*

### ⑤ 성찰 + 과정 증거 (Reflect + Evidence) — 교사 인정 자산
- **무엇:** 무엇을 왜 어떻게 고쳤는지 학생 자기설명 + **과정 로그**(초안 이력, 코치가 한 것/안 한 것).
- **동작:** 드래프트 타임라인 + "AI 사용 투명 기록"(2026 교육부 가이드 요건 충족) → 교사·학부모가 "직접 썼다"를 확인.
- **차별점:** 경쟁사 미보유. 유통(교사 인정)·신뢰의 핵심 산출물.

## 4. "대신 안 써준다"의 기술 설계 (신뢰의 엔진)

> 이게 무너지면 ChatGPT와 같아지고 교사 인정·자유 사용이 동시에 깨짐. 핵심 IP.

- **생성 차단 가드:** 코치 출력은 *질문·진단·발판*만 허용. "학생 글을 대신 작성/문장 완성"은 시스템 프롬프트·후처리에서 차단(현 `app/lib/grading.ts` 가드 패턴 확장).
- **메타인지 프롬프트:** 답 대신 "왜?/근거는?/반대 입장은?/네 경험은?"으로 사고를 외화.
- **루브릭 정렬:** 교사/과제 루브릭을 1급 입력으로 — 코칭이 *그 기준*에 묶임(범용 LLM이 못 하는 것).
- **과정 무결성:** 드래프트 버전·편집 이력 서버 기록 → 위변조 어려운 "직접 썼다" 증거.
- **엔진 공유:** 위 모듈은 순수 로직으로 분리해 **주니어(초등) 글쓰기 모드와 공유**(연령별 질문·톤·UX만 분기).

## 5. UI/UX 방향 (사용성 약점 정조준)

- **레이아웃 = "캔버스 + 여백 코치"** (챗봇 입력창 ❌): 왼쪽 글 쓰는 캔버스, 오른쪽/여백에 코치 발판·질문. 글이 주인공, 코치는 옆에서 거듦.
- **모바일·태블릿 우선:** 중·고생 현실은 폰. 캔버스·여백 코치를 좁은 화면용으로(탭 전환/시트). PDF·사진 과제 업로드 모바일 최적.
- **성장 가시화:** 홈에 5영역 시계열 + "이번에 늘어난 것" 배지 + 고쳐쓰기 전/후 나란히. *성장이 제품의 감정.*
- **톤:** 주니어 푸리와 달리 **담백한 코치 톤**(중고생은 유아틱 거부). 격려는 하되 과장 금지.
- **마찰 최소:** ②~④는 질문·발판이 *짧게* 끊겨 나와 한 호흡에 한 가지만(과부하·이탈 방지).
- **OS 셸 정합(락):** blue+lemon 등 OS 토큰 준수. ※radius 토큰은 이 레포에 아직 없음(globals.css=band/accent/surface만) — 출처 확인 후 정합(`--r-*`는 pullim-web 것). 단품이지만 OS 부품으로 보일 수 있게.

## 6. 교사 인정 설계 (유통이자 신뢰)

> 결정 6(교사 인정 주도 유통)의 제품 측 구현. "교사 도구"를 따로 파는 게 아니라, 학생 제품에 *교사가 인정할 근거*를 내장.

- **교사용(무료 진입):** ① 루브릭 등록(코칭이 내 기준에 맞춤) ② 반 학생 과정 로그 열람("누가 직접 썼나, 어디서 막혔나") ③ "AI 사용 투명" 리포트. **채점 자동화는 안 만듦**(정부 무료 채움아이·KEEwi와 경합 회피) — 교사가 원하는 건 대필 적발이 아니라 *학생이 진짜 성장했다는 증거*.
- **루프:** 교사가 "이건 가르치는 도구"로 인정·추천 → 학생 자유 사용 → 과정 로그가 다시 교사 신뢰 강화.

## 7. GTM — 교사 인정 주도

| 단계 | 경로 | 깔때기 |
|---|---|---|
| **1. 교사 시딩(0~3M)** | 개별 교사에 무료 도구(루브릭+과정로그) 배포(쌤동네 30만·교과 커뮤니티) | 교사 인정·반 단위 사용 → 학생 유입, CAC↓ |
| **2. 학생 자유 사용 → 가계 전환** | 교사 인정 받은 학생이 집에서도 사용 | 무료 코칭 체험 → 학부모 구독 |
| **3. 강사·DTC 보조** | 자율 논술강사 시딩 / 커뮤니티 | before/after 후기 자산 |

**메시지:** "성적 오른다" ❌ → **"대신 안 써주고, 네가 더 잘 쓰게 — 교사도 인정하는 글쓰기 코치."** (효과 통계는 쌓이면 보조로, 깃발은 성장·신뢰)

- ⏳ 가격 확정은 WTP 검증(Van Westendorp + 대체/추가 의향) 후 🔒.

## 9. 컴플라이언스 (게이트, 접어두되 처음부터)

- **만 14세:** 만14세+(중2~고3) 본인 동의 우선, 중1(만13) 법정대리인 별도 트랙([개인정보보호법 §22의2](https://casenote.kr/법령/개인정보_보호법/제22조의2)).
