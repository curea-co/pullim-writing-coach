## 0. 한 줄 요약

30일 안에 학생이 수행평가 글을 넣고 5영역 채점·첨삭·수정 방향을 즉시 받는 한국어 글쓰기 코치 MVP. 한우리(월 17~43만원, 사람 첨삭, 수행평가 비정렬) 대비 **가격 1/10 · 즉시성 · 2022 개정 교육과정 정렬** 세 축으로 진입한다. 본 문서는 14개 기능 후보를 점수화하고, 7개 이하로 컷한 **P0 배포 기능 7개**를 제안한다.

## 1. 의사결정 컨텍스트

### 1.1 30일 정책 제약 (상위 문서)

`choi_sunhye_employee_guideline_30day_launch_2026-05-13.md` §5에서 정의된 30일 내 필수 9개 동작:

```text
1. 사용자가 과제 설명을 입력한다.
2. 사용자가 본문 글을 붙여넣는다.
3. AI가 5영역 점수를 낸다.
4. AI가 문장/구조/논리 피드백을 준다.
5. AI가 수정 방향을 제시한다.
6. 결과 화면을 볼 수 있다.
7. 결과를 복사하거나 저장할 수 있다.
8. 배포 URL이 있다.
9. desktop/mobile 화면 증거가 있다.
```

§13 최종 지시는 **"Writing Coach를 30일 안에 배포"** 하나로 좁혀져 있고, §9 09:30 Work Contract는 "30일 배포 MVP 기능 목록 **7개 이하**"를 명시한다. 따라서 본 제안서의 컷 기준은 **"위 9개 동작을 7개 이하의 기능 카드로 묶어 배포한다"** 이다.

### 1.2 시장 진입점 (benchmarking_2026-05-12.md)

한우리(독서토론논술) 실측 결과 — AI 진입 여지 3가지:

| 진입 축 | 한우리 현 상태 | Writing Coach 목표 |
|---|---|---|
| 가격 | 월 17.3~43.2만원 (학원형, 강남 2024-07) | 월 2.9~5.9만원 (1/5~1/10) |
| 즉시성 | 주 1회 회수 첨삭 (수기) | 입력 → 결과 10초 이내 |
| 수행평가 정렬 | 자체 도서 기반, 학교 양식 비정렬 | 2022 개정 성취기준 + 학교 양식 정렬 |

세 진입 축은 본 제안서의 **차별화 기여도** 평가 기준이 된다.

### 1.3 교육과정 기반 (2022_curriculum_writing_standards.md)

2022 개정 국어과 교육과정에서 추출된 **쓰기 산출물 관련 성취기준 54개**. 산출물 유형 분포:

- 설명문 / 보고서 (정보 전달)
- 논설문 / 논증문 / 토론 입론문 (주장)
- 성찰문 / 수필 (정서·경험)
- 요약문 / 독후감 / 서평 / 비평문 (읽기 기반)
- 매체 자료 / 콘텐츠 (복합양식)

이 분포는 **장르별 rubric 가중치 분기**의 근거가 된다 (v1 검토 사항, 별도 기능 카드 아님).

## 3. 기능 후보 14개

### F01. 과제 조건 입력 폼 (과목·학년·장르·분량·제출일)

- **사용자 가치**: 학생이 자기 과제 맥락을 1분 안에 AI에 전달. 입력 표준화로 첨삭 품질 편차 ↓.
- **30일 적합성**: Week 1~2 가능. 폼 자체는 단순.
- **차별화**: 학교 양식·과목별 채점 관행 정렬의 입구. 한우리는 도서 기반이라 없음.
- **복잡도**: S
- **점수**: 3 + 3 + 3 + 3 = **12 / P0**

### F02. 학생 글 본문 입력 (붙여넣기 + 글자 수)

- **사용자 가치**: 핵심 입력. 글자 수 카운터로 분량 기준 만족 자기 확인 가능.
- **30일 적합성**: Week 1 가능.
- **차별화**: 평범. 없으면 서비스 자체 성립 X.
- **복잡도**: S
- **점수**: 3 + 3 + 1 + 3 = **10 / P0**

### F03. AI 5영역 점수 산출 (각 20점, 총 100점)

- **사용자 가치**: 학생·학부모가 한눈에 "지금 몇 점짜리"인지 인지. 비교·재제출 동기.
- **30일 적합성**: rubric v0.1이 Week 1 산출물. 가능.
- **차별화**: 한우리는 점수 비공개 — 본 항목 자체가 우위.
- **복잡도**: M (rubric 정확도 + 프롬프트 튜닝 비용)
- **점수**: 3 + 3 + 3 + 2 = **11 / P0**

### F04. 영역별 문장 단위 피드백 (문장/구조/논리)

- **사용자 가치**: "왜 12점인지"의 근거. 학생이 다음 수정에 바로 반영 가능 (rubric §개선가능성과 연결).
- **30일 적합성**: 프롬프트 v0.2~v0.3 단계. Week 2~3.
- **차별화**: 한우리 사람 첨삭의 핵심 가치를 AI로 복제하는 지점 — 이게 안 되면 진입 불가.
- **복잡도**: M (문장 단위 매핑 UI + 프롬프트 일관성)
- **점수**: 3 + 3 + 3 + 2 = **11 / P0**

### F05. 수정 방향 제안 (다음 1~2개 액션)

- **사용자 가치**: "그래서 뭐부터 고쳐?"의 답. 학습 효과의 종착지.
- **30일 적합성**: Week 2 가능. 프롬프트 추가 1개.
- **차별화**: 한우리 회수 첨삭은 다음 수정 사이클이 1주. 즉시성 우위 직결.
- **복잡도**: S~M
- **점수**: 3 + 3 + 3 + 2 = **11 / P0**

### F06. 결과 화면 (점수 + 피드백 + 수정 방향 통합 뷰)

- **사용자 가치**: 한 화면에서 점수→근거→액션의 흐름. 인지 비용 최소화.
- **30일 적합성**: Week 2 staging에 필수.
- **차별화**: 화면 자체는 평범하나, 없으면 §5 30일 필수 6번 미충족.
- **복잡도**: M (디자인 + 반응형)
- **점수**: 3 + 3 + 1 + 2 = **9 / P0**

### F07. desktop / mobile 반응형 화면

- **사용자 가치**: 학생의 1차 디바이스는 모바일. 학부모는 desktop도 사용.
- **30일 적합성**: Week 2~3. CSS 작업.
- **차별화**: 평범. §5 30일 필수 9번 직결.
- **복잡도**: M
- **점수**: 3 + 3 + 0 + 2 = **8 / P1** (단, §5 필수이므로 P0로 강제 승격)

### F08. 결과 복사 / 저장 (텍스트 클립보드 + 이미지 PNG 1종)

- **사용자 가치**: 학생이 노트·과제 파일에 붙여넣기. 학부모·교사 공유.
- **30일 적합성**: Week 3~4. 라이브러리 사용 시 단순.
- **차별화**: 평범. §5 30일 필수 7번 직결.
- **복잡도**: S
- **점수**: 3 + 2 + 1 + 3 = **9 / P0**

### F09. 학교급·과목 프리셋 (중2 국어 / 중3 사회 / 고1 국어 …)

- **사용자 가치**: F01 입력을 3초로 단축. 신규 학생 진입 마찰 ↓.
- **30일 적합성**: Week 2~3. 데이터는 §1.3 교육과정 표에서 추출.
- **차별화**: 수행평가 정렬의 가시적 신호. 한우리는 도서 단위라 없음.
- **복잡도**: S~M
- **점수**: 2 + 2 + 3 + 3 = **10 / P0 후보** (단, 7개 컷에서 가장 양보 가능)

### F10. 교육과정 과목·영역 단위 태깅 (공통국어 / 화법과 언어 / 독서와 작문 / 문학 등)

- **사용자 가치**: 학생·학부모·교사에게 "이 글이 2022 개정 교육과정의 어느 과목·영역에 닿는가" 명시. 성취기준 코드보다 거칠지만 직관적이라 신뢰 신호로 충분.
- **30일 적합성**: Week 3~4. 분류 자체는 단순하나 결과 화면 통합 작업이 후반.
- **차별화**: 본 서비스 고유의 진입 축. 한우리·대교·과외 모두 미보유.
- **복잡도**: S~M (6~9개 과목·영역 LLM 분류, 102개 성취기준 코드 매핑 불필요)
- **점수**: 2 + 2 + 3 + 3 = **10 / P1** (P0 컷 7개 후순위 — Week 4 추가 시도)

### F12. 영역별 0~20점 구간 설명 (왜 12점인지)

- **사용자 가치**: 점수의 설명 가능성. 학부모 신뢰 직결.
- **30일 적합성**: Week 1~2 rubric 작성과 동시.
- **차별화**: 한우리 점수 비공개 대비 강한 우위.
- **복잡도**: S (rubric 텍스트 작업)
- **점수**: 3 + 3 + 2 + 3 = **11 / P0**

### F13. 수정 전/후 비교 보기 (1차 글 vs 재제출 글)

- **사용자 가치**: 학습 곡선 가시화. 학부모 결제 후 효과 체감 핵심.
- **30일 적합성**: Week 4 빠듯. 저장·세션 필요.
- **차별화**: 명확한 우위. 한우리는 종이 회수 구조라 비교 어려움.
- **복잡도**: L (세션·이력 저장 + diff UI)
- **점수**: 1 + 3 + 3 + 1 = **8 / P1** (Month 2)

### F14. 교사용 / 학부모용 / 학생용 리포트 톤 분리

- **사용자 가치**: 같은 결과를 3개 톤으로. 학부모는 결제자, 교사는 채택 신호.
- **30일 적합성**: Week 1~2 샘플은 만들지만, 화면 전환 UI는 Week 4 빠듯.
- **차별화**: 한우리는 학부모 리포트만 있고 학생/교사 톤 분리 없음.
- **복잡도**: M (프롬프트 3종 + 화면 탭)
- **점수**: 2 + 3 + 2 + 2 = **9 / P0 후보** (7개 컷에서 학생용 1종만 v1, 나머지는 Month 2)

### F15. 익명 1회 무료 체험 (회원가입 없이)

- **사용자 가치**: beta 사용자 10명 확보의 마찰 제거. §10 평가 기준 직결.
- **30일 적합성**: Week 3~4 가능. 쿠키·세션 기반.
- **차별화**: 한우리는 상담 후 가입. 본 서비스 즉시성 신호.
- **복잡도**: M (남용 방지·IP rate limit)
- **점수**: 2 + 2 + 2 + 2 = **8 / P1**

§9 09:30 Work Contract의 "7개 이하" 제약을 만족시키기 위해 **기능 카드 단위로 묶어** 7개로 컷한다.

| # | MVP 기능 카드 | 묶은 후보 | Week |
|---:|---|---|---|

| 2 | **5영역 점수 + 구간 설명** | F03 + F12 (100점 만점, 각 영역 0~20점 + 왜 그 점수인지) | W1~W2 |

| 7 | **학생용 리포트 톤** | F14 일부 (학생용 1종만, 교사·학부모용은 Month 2) | W3~W4 |

**컷 근거**:
- 1~7번은 §5 30일 필수 9개 동작을 모두 커버한다 (매핑은 §6.2 참조).
- F09 학교급 프리셋은 카드 1에 흡수 — 별도 카드로 두면 7개 초과.
- F10 과목·영역 태깅 / F13 수정 이력 / F15 익명 체험은 **Month 2 최우선 후보**로 분리.
- 장르별 rubric 가중치 분기는 v0.1에서 단일 가중치로 출발, v1에서 분기 도입 검토 (별도 기능 카드 아님).

### 5.1 Writing Coach 기능 구조 (계층별 매핑)

확정된 7개 화면 기능을 §3 F-코드에 매핑하고, 구현 계층(기술·UX·UI·서비스정책)으로 분류한다. 위 §5의 7개 MVP 카드가 "30일 배포 단위" 뷰라면, 본 구조는 "구현 계층 단위" 뷰 — 같은 기능 집합을 다른 축으로 본 것이다.

**[기술 계층] 확정 7개 화면 기능 ↔ F-코드**

| # | 화면 기능 (확정) | 매핑 F-코드 | 비고 |
|---:|---|---|---|
| 1 | 과제 설명 입력 | F01 과제 조건 입력 폼 · F09 학교급·과목 프리셋 | F09는 F01 입력을 3초로 단축 |
