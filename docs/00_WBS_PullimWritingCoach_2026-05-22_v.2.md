## 4. 의존성·임계경로

```text
[P-1.1 채점 철학 결정] ──(범위 확정)──┐
                                      ▼
0.3 프롬프트 v0.2 → 1.1 재검증(eval 게이트) → 2.1 /api/score → 2.4 BE 테스트 → [M1]
   → 3.2 FE 라이브 연동 → 3.4 통합 리뷰 → 4.x → [M2]
   → 5.x 테스트·픽스 → 6.x 배포 → [M3]
```

**진짜 임계경로 = P-1.1 채점 철학 결정 → 그 다음이 프롬프트 v0.2 보정.** 09는 모델이 *틀린* 게 아니라 EPO보다 *박함*(분산은 안정적)을 보였다. 그래서 "모델 62를 기준으로 한다"는 결정이 나면 캘리브레이션은 거의 불필요(이미 self-consistent)하고 일정은 여유롭다. "EPO 86에 맞춰라"면 few-shot이 핵심 난제. **그래서 이 결정을 05-22 즉시 기준 B·기준 A에 맞춰 본인이 내려 05-26 캘리브레이션 시작 전 확정한다(외부 회신 대기 없음).** M1은 'BE 골격 시연'(엔드투엔드 1건)으로 정의했으므로 05-28 충족 가능, 완전 보정·5종 통과는 M1 이후 + 휴일 보완.
