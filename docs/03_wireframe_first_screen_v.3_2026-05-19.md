## 0. 화면 원칙

- **단일 페이지**: 입력 → 실행 → 결과가 한 화면에서 스크롤로 이어진다. 페이지 이동 없음.
- **위에서 아래로 1방향**: 학생이 헷갈리지 않게 흐름을 세로 1줄로 고정.
- **모바일 우선**: 학생 1차 디바이스는 모바일. desktop은 같은 구조를 넓게.
- **문구는 학생 언어**: 존댓말, 짧게, 지시가 아니라 안내.

## 1. 화면 영역 구조 (3블록)

```text
┌─────────────────────────────────┐
│  블록 A. 헤더                    │
├─────────────────────────────────┤
│  블록 B. 입력 영역               │
│   B1. 과제 설명 입력 (F1)        │
│   B2. 학생 글 입력 (F2)          │
│   B3. 실행 버튼 (F3)             │
├─────────────────────────────────┤
│  블록 C. 결과 영역 (F3 실행 후)  │
│   C1. 총점 + 5영역 점수 (F4)     │
│   C2. 영역별 피드백 (F5)         │
│   C3. 수정 가이드 (F6)           │
│   C4. 결과 복사 (F7)             │
│   C5. 면책 문구                  │
```

- 블록 C는 F3 실행 전에는 화면에 없음 (또는 비활성 안내만).
- 블록 C가 나타나면 화면이 자동으로 C1 상단으로 스크롤.

## 4. 결과 영역 구성 (C1~C5 상세)

### C1. 점수 (F4)

```text
┌──────────────────────────────┐
│  점수                         │
│                              │
│        72 / 100              │   ← 총점, 가장 큰 텍스트
│                              │
│  과제 이해      16 / 20  ▓▓▓▓░ │
│  내용 충실도    14 / 20  ▓▓▓░░ │
│  구조·논리      15 / 20  ▓▓▓▓░ │
│  표현·문장      14 / 20  ▓▓▓░░ │
│  성장 가능성    13 / 20  ▓▓▓░░ │   ← v0.3 영역명
└──────────────────────────────┘
```

영역 편차 6점 이상일 때 (예: 06 v.3 샘플 C 시계 본문 — §3.2 17 / §3.3 9):

```text
┌──────────────────────────────┐
│  점수                         │
│                              │
│        61 / 100              │
│  ⚠  영역 편차가 큽니다 —      │   ← v0.3 신설: 편차 6점 이상 안내
│     총점보다 영역별 피드백을  │      (시각 표현은 Week 2 design)
│     먼저 보세요               │
│                              │
│  과제 이해      14 / 20  ▓▓▓░░ │
│  내용 충실도    17 / 20  ▓▓▓▓▓ │   ← 가장 높은 영역 강조
│  구조·논리       9 / 20  ▓▓░░░ │   ← 가장 낮은 영역 강조
│  표현·문장     10 / 20  ▓▓░░░ │
│  성장 가능성    11 / 20  ▓▓░░░ │
└──────────────────────────────┘
```

- 총점: 화면 최상단, 가장 큰 글자.
- 5영역: 영역명 + 점수 + 막대. 막대 색은 0~9 / 10~14 / 15~20 3단계 (색상값 Week 2 design).
- **영역 편차 6점 이상**(`max(scores) - min(scores) >= 6`): 총점 바로 아래에 ⚠ 안내문 노출 + 최고·최저 영역 막대를 시각적으로 강조 (functional_spec §3 F4 v0.3 신설).
- `functional_spec §4.2 scores` 배열 순서 고정.

### C2. 영역별 피드백 (F5)

```text
┌──────────────────────────────┐
│  영역별 피드백                 │
│                              │
│  ▸ 과제 이해                  │
│    잘한 점  ...               │
│    고칠 점  ...               │
│                              │
│  ▸ 내용 충실도                │
│    잘한 점  ...               │
│    고칠 점  ...               │
│                              │
│  ▸ 구조·논리                  │
│  ▸ 표현·문장                  │
│  ▸ 성장 가능성                │   ← v0.3 영역명
└──────────────────────────────┘
```

- 영역 5개 각각: "잘한 점" 1~2문장 + "고칠 점" 1~2문장.
- `feedback_fix` 한 문장에 'A→B' 정정 항목이 4개 이상이면 일부는 C3 수정 가이드로 분리 (functional_spec §3 F5 v0.3 신설 룰).
- "잘한 점"과 "고칠 점"은 라벨로 시각 구분 (색/아이콘은 Week 2).

### C3. 수정 가이드 (F6)

```text
┌──────────────────────────────┐
│  이렇게 고쳐 보세요             │
│                              │
│  1  본론에 근거를 1개 더 추가  │
│     → 과제가 요구한 근거 2개 ..│
│                              │
│  2  본론 2문단을 사례로 교체   │
│     → 같은 근거가 반복되어 ... │
│                              │
│  3  '것 같다'를 단정형으로 ... │
│     → 주장하는 글은 분명한 ... │
└──────────────────────────────┘
```

- 1~3개 카드, priority 순서.
- 각 카드: 번호 + 실행형 한 줄 + "→" 근거 한 줄.

### C4. 결과 복사 (F7) + 재제출

```text
┌──────────────────────────────┐
│   [ 결과 복사하기 ]            │
│   [ 글 고치고 다시 받기 ]       │
└──────────────────────────────┘
```

- "결과 복사하기": `functional_spec §5` 텍스트 포맷 클립보드 복사 → "복사됨 ✓" 토스트.
- "글 고치고 다시 받기": 화면을 블록 B로 스크롤 (입력값 유지, attempt_no +1).

### C5. 면책 문구

```text
※ 이 채점은 AI 자동 채점입니다.
  학교 교사의 실제 채점과 다를 수 있습니다.
```

- 결과 영역 맨 아래 고정. 복사 텍스트에도 포함 (§5 참조).

## 7. 마이크로카피 모음 (상태 메시지)

| 상황 | 화면 문구 |
|---|---|
| 첨삭 진행 중 | AI가 글을 읽고 있어요… |
| 복사 성공 | 결과가 복사되었어요 |
| 복사 실패 (E7) | 복사에 실패했어요. 결과 영역을 직접 선택해 복사해 주세요 |
| 본문 부족 (E2) | 본문을 50자 이상 입력해 주세요 |
