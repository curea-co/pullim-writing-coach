## 0. 검수 범위·기준

본 검수는 06 v.3 파일이 만든 F3 출력 3건(rubric v0.4 적용)에 대한 **PM 수동 검수**다. EPO 관점의 4축으로 본다.

#### A-1. 권고 (분량 표기 정밀도)

과제 이해 `feedback_fix`: "목표 500자 중 **약 280자**로 분량이 부족해요." → 입력 JSON `char_count`가 **281**로 명시되어 있고, 06 v.3 본문 §1.A·§2.A·§3.2 표 모두 281을 사용한다. 학생용 출력만 '약 280자'로 반올림돼 일관성이 깨진다.

- **(a) "281자"로 정확히 표기**(권장) — 분량 안내는 학생이 '얼마를 더 써야 하나'를 가늠하는 수치이므로 정확값이 신뢰에 유리
- (b) 유지 — '약 N자'는 학생 부담을 낮춘다는 코칭 톤 측면에서 정당화 가능

EPO 추천: **(a)** 채택. 1자 단위가 어색하면 "281자(목표의 56%)" 식 비율 병기도 검토.

#### A-2. 권고 (fix 길이 — UX)

과제 이해 fix("목표 500자 중 약 280자로 분량이 부족해요. 각 화산의 형성 과정이나 특징을 한 문장씩 더 적어 보세요.")는 약 50자 두 문장으로 functional_spec §3 F5 "60자 이내 권장" 안에 있다. 다만 **혼합 메시지**(분량 안내 + 액션)다. 학생용 카드에서 한 메시지 한 호흡 원칙이라면:

- **분량 안내는 영역 fix가 아니라 상단 메타 영역**(예: 진행 막대 옆)에 두고, fix는 액션만으로 압축하는 안을 functional_spec §3 F4 화면 가이드 갱신 시 검토.

본 출력 자체는 채택 — 권고는 화면 가이드 차기 개정 검토 사항.

#### B-1. 권고 (영역 경계 — '구가간')

내용 충실도 fix가 "'구가간'은 '구간'의 오기"로 시작한다. 동시에 표현·문장 fix에는 '이글'·'뿐 더러'·누락 마침표 3곳이 묶였는데 **'구가간'은 빠져 있다**. 영역 경계(rubric §2 원칙 4) 관점에서:

- '구가간 → 구간'은 **오탈자**(mechanics)이므로 §3.4 표현·문장이 1차 영역
- 단 EEZ 정의문 한가운데에 들어가 정보 정확도를 흐린다는 측면에서 §3.2 내용 충실도 부속 약점으로 볼 여지도 있음(rubric §2.1 '설명문' 행 '정확한 정보')

옵션:
- **(a) §3.4 fix에 합쳐 4곳 묶기** — 단 본 권고는 C-1 수정 이슈(정정 4건 묶음 금지)와 충돌. 묶을 경우 4건 중 1건은 revision_guides로 분리 필요
- **(b) §3.2 fix를 다른 내용 약점으로 교체** — 예: "후반부 객관 정보(영해/EEZ)와 동기 진술이 한 단락에 섞여 있어 정보의 결이 일부 흐려져요" 등 §3.2 본연의 '근거의 다층위' 축으로 재작성, '구가간'은 §3.4 fix에 통합(3건 묶음으로 룰 위반 없음)
- (c) 유지 — '구가간'의 정보 정확도 측면을 인정해 §3.2에 둠

EPO 추천: **(b)**. §3.2 18은 내용 우수 등급인데 fix가 단일 정정으로 끝나면 학생이 '내용 영역에서 받을 액션'을 놓친다. mechanics는 §3.4로 몰아주고 §3.2 fix는 내용 영역의 다음 도약 지점(예: 결론 객관 정리·EEZ 권리/제한의 추가 사례)으로 채우는 편이 학생·학부모 가치에 더 부합.

#### C-1. 즉시 수정 (rubric §2 원칙 12 + §5.2 위반)

`scores[3].feedback_fix` (표현·문장):

```text
"'~게 된다'가 본문 전체에서 열 번 넘게 반복돼요. 일부를 '~한다 / ~된다 / ~다'로
바꿔 보세요. 그리고 '가김기→감기고', '초짐→초침', '막았던게→막았던 게',
'뿐 더러→뿐더러'도 함께 고쳐 주세요."
```

위반 사항 2건:

1. **rubric v0.4 §2 원칙 12**: 한 fix에 'A→B' 형식 정정 항목 4건이 묶임(가김기·초짐·막았던게·뿐 더러). 룰은 3개 이하.
2. **rubric v0.4 §5.2**: 동일한 4건이 `revision_guides[2]`에도 그대로 등장 — 중복 메시지 방지 룰 위반.

본 룰은 **이번 v0.4에서 신설**됐다(`06 v.3 §4-5` 검수 메모 → rubric v0.4 §1 #6). 즉 06 v.3은 rubric v0.4를 적용했다고 선언했으나, 정작 v0.4가 신설한 룰을 §3.4 fix에서 어겼다 — **rubric/출력 정합성 결함**. 학생용 카드 UX 60자 가이드 위에 빽빽한 정정 4건이 얹히면 학습 부담이 급증한다.

**수정안** (rubric §5.2 배치 원칙 — "두 곳에 모두 해당할 때 revision_guides에 우선, 영역 fix는 다른 약점으로"):

```diff
- "feedback_fix": "'~게 된다'가 본문 전체에서 열 번 넘게 반복돼요. 일부를 '~한다 / ~된다 / ~다'로 바꿔 보세요. 그리고 '가김기→감기고', '초짐→초침', '막았던게→막았던 게', '뿐 더러→뿐더러'도 함께 고쳐 주세요."
+ "feedback_fix": "'~게 된다'가 본문 전체에서 열 번 넘게 반복돼요. 일부를 '~한다 / ~된다 / ~다'로 바꿔 보세요."
```

`revision_guides[2]`("'가김기/초짐/막았던게/뿐 더러' 네 곳을 정정하세요.")는 그대로 두면 학생 우선순위 액션으로 한 번만 등장 — 룰 정합·UX 모두 충족.

> **AI 첨삭 프롬프트 v0.1에 강제 룰로 못박을 것**: rubric v0.4 §5.3 POSITION_CHECK 옆에 동일 형식의 [FIX_COUNT_CHECK] 추가. 출력 직전 각 영역 `feedback_fix`의 'A→B' 패턴을 세어 4개 이상이면 자동으로 마지막 N-3개를 `revision_guides`로 이동시키는 후처리 룰. §5 후속 작업으로 등록.

#### C-2. 권고 (fix 길이 — UX)

과제 이해 fix("글 전체가 한 흐름으로 흘러 '단계별'이 시각적으로 안 드러나요. 단계마다 줄을 바꿔 번호나 소제목을 붙여 보세요. 분량도 600/800자라 한 단계 정도 더 풀어 설명할 여유가 있어요.")는 3개 문장·약 100자로 functional_spec §3 F5 "1문장 60자 이내 권장"을 초과한다.
