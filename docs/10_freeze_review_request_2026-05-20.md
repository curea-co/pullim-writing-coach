### 1.3 이번 freeze의 직접 트리거 — 06 v.4 패치 (07 v.2 반영)

`07_pm_review_v.2`가 06 v.3 출력을 검수해 **즉시 수정 1건 + 권고 4건**을 냈고, 이를 06 v.4에 모두 반영했다(§4 상세). 핵심은 **C-1**: rubric §2 원칙 12(fix 정정 3개 이하)를 같은 날 작성된 출력이 어긴 정합성 결함 → 06 v.4에서 수정 + 08 프롬프트에 강제 룰로 못박음.

### 1.4 두 기준으로 판정할 항목

- **기준 A**: 각 산출물의 채택/수정/기각 (특히 미검증 항목을 숨기지 않았는지)
- **기준 B**: 학생·학부모 가치·톤·학년 적절성, 데모 화면 작동, 60자 룰 등 열린 의제

### 1.5 freeze 전 반드시 짚어야 할 열린 항목 (두 기준으로 판정)

1. **AI 프롬프트 v0.1 재현 검증 실행 완료 (2026-05-21, `09_prompt_verification`) — 결과: HARD PASS ❌, freeze 보류 권고.** 08 §6의 5건×5회=25건을 실호출(claude-sonnet-4-6, temp 0.2)로 돌린 결과: **재현성·스키마·FIX_COUNT(C-1 회귀 방지)는 PASS**이나, **EPO 정답 점수를 재현 못 함**(상위권 체계적 하향: B −24·E −13·A −12, 고품질 글이 ~62로 수렴)이고 **응답시간 ~25초로 15초 목표 초과**. 즉 '일관되지만 EPO보다 박하다'. → 권고: **프롬프트 v0.2(few-shot 캘리브레이션) 보정 후 재검증을 freeze 전제로**, 또는 **rubric·샘플은 freeze하되 프롬프트는 v0.2까지 별도 트랙** 분리. ⚠️ 핵심 결정: **A 74·B 86(EPO)이 옳은가, 모델의 62가 옳은가** — 기준 B·기준 A에 맞춰 EPO가 채점 철학(상위 구간 후함)을 판정. (상세 §4 / `09`)
2. **점수대 슬롯 2개 부재** — 절대 저점(0~34)·완성(90~100). 의도적 미보강(§6 한계) vs 샘플 F·G 추가.
3. **functional_spec §3 F5 '60자' 룰** — 1문장 단위인지 fix 전체인지, 권장인지 hard cap인지 (박승훈).
4. **§5.4 저점 fix 톤** — 코칭 톤이 면죄부가 되지 않는지 (박승훈).
5. **강제 룰 보장 수준** — 프롬프트만으로 충분한지, 클라이언트 후처리 가드가 freeze 조건인지.
