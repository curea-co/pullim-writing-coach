## 2. v1 스키마 (PostgreSQL 가정)

### 2.1 `users` — 공통

| 컬럼 | 타입 | 제약 |
|---|---|---|
| `id` | uuid | PK |
| `email` | citext | UNIQUE NOT NULL |
| `password_hash` | text | NULL 허용 (OAuth-only 가입자) |
| `role` | enum('student','teacher') | NOT NULL |
| `display_name` | text | NOT NULL |
| `email_verified_at` | timestamptz | NULL |
| `status` | enum('active','suspended','deleted') | NOT NULL DEFAULT 'active' |
| `created_at` / `updated_at` | timestamptz | NOT NULL DEFAULT now() |
| `last_login_at` | timestamptz | NULL |

### 2.2 `oauth_accounts` — OAuth 연결

> ⚠️ 인증 라이브러리 자체 테이블로 대체될 자리 (NextAuth/Auth.js·Supabase Auth 등은 자체 `accounts`·`sessions`·`verification_tokens`를 강제). 라이브러리 미정 동안의 placeholder.

| 컬럼 | 타입 | 제약 |
|---|---|---|
| `id` | uuid | PK |
| `user_id` | uuid | FK→`users.id` NOT NULL, INDEX |
| `provider` | enum('google','kakao') | NOT NULL |
| `provider_account_id` | text | NOT NULL |
| `created_at` | timestamptz | NOT NULL DEFAULT now() |
| UNIQUE(`provider`, `provider_account_id`) | | |

### 2.3 `student_profiles` — 1:1 with users where role='student'

| 컬럼 | 타입 | 제약 |
|---|---|---|

| `birth_year` | smallint | NOT NULL (만 14세 미만 판정용) |

### 2.4 `teacher_profiles` — 1:1 with users where role='teacher'

### 2.8 `share_tokens` — 학생이 발급한 채점 결과 공유 토큰 ((c) 정책 잠금)

| 컬럼 | 타입 | 제약 |
|---|---|---|
| `id` | uuid | PK |
| `scoring_record_id` | uuid | FK→`scoring_records.id` NOT NULL, INDEX |
| `code` | text | UNIQUE NOT NULL (URL-safe 16+자, secure random) |
| `created_by_user_id` | uuid | FK→`users.id` NOT NULL (발급 학생, 소유 검증용) |
| `label` | text | NULL (학생 메모, 예: "박선생님께") |
| `expires_at` | timestamptz | NULL (미설정=무기한) |
| `revoked_at` | timestamptz | NULL (학생이 폐기 시) |
| `last_accessed_at` | timestamptz | NULL (마지막 조회 시각) |
| `access_count` | int | NOT NULL DEFAULT 0 (누적 조회 수) |
| `created_at` | timestamptz | NOT NULL DEFAULT now() |
| INDEX(`scoring_record_id`), INDEX(`created_by_user_id`) | | |
| CHECK (`revoked_at` IS NULL OR `revoked_at` >= `created_at`) | | |

## 3. 가입 트랜잭션 흐름 (단일 TX)

```
1) users INSERT
2) role별 *_profiles INSERT (학생이면 birth_year 포함)
3) user_consents INSERT (privacy_required 최소 1건 필수)
4) students.birth_year 기준 만 나이 < 14
   → guardian_consents INSERT 필수
   → 누락 시 ROLLBACK (앱 레벨 가드 + DB CHECK/트리거 보조 권장)
```

---

## 4. 정책 — 앱 레벨에서 강제 (DB 보조 OK)

### 4.1 토큰 공유 (share_tokens)

1. **발급**: 학생이 본인의 `scoring_record`에 대해서만 토큰 생성
   (`created_by_user_id` = `scoring_records.student_user_id`).
2. **조회 자격**: 토큰으로 접근 가능한 사용자는 **role='teacher'인 로그인 사용자만**.
   - 비로그인·학생 role은 거부 (미성년 학생 데이터 보호).
   - 자격 통과 시 `last_accessed_at` = `now()`, `access_count` += 1.
3. **유효성**: `revoked_at IS NULL AND (expires_at IS NULL OR expires_at > now())`일 때만 통과.
4. **다중 발급**: 동일 `scoring_record`에 여러 토큰 허용 (교사별 별도 코드 → 누가/어느 코드로 봤는지 식별).
5. **불변**: `scoring_record_id`·`created_by_user_id`는 immutable.

### 4.2 미성년 보호자 동의

- 회원가입 화면에서 `birth_year` 입력 → 만 14세 미만이면 보호자 동의 화면 분기.
- 동의 완료 전엔 `users.status` = `'pending'` 또는 트랜잭션 보류.

### 4.3 동의 철회

- 회원이 `user_consents.revoked_at`을 채울 수 있음.
- `privacy_required` 철회 시: 계정 비활성(`users.status` = `'deleted'` 소프트 삭제) + 개인정보 익명화·삭제 흐름 별도(법적 보존기간 고려).

## 5. 보안 노트 (앱 레벨)

- `share_tokens.code` 생성: `crypto.randomBytes(16).toString('base64url')` 권장 (≥22자). UNIQUE 충돌 시 재시도.
- 토큰 조회 엔드포인트에 **rate-limit** 권장 (브루트포스 차단).
- 학생용 "내 공유 코드 목록/폐기" UI 필요 (`revoked_at` 토글).
- `password_hash`는 bcrypt/argon2 등 표준 KDF만 사용 (인증 라이브러리가 보장).
- 모든 PII 컬럼은 백업·로그에서 마스킹 정책 별도.

## 7. 보류된 3개 결정이 정해지면 영향받는 부분

| 보류 항목 | 영향받는 자리 |
|---|---|
