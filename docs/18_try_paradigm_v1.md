# /try 패러다임 v1 — 설계 메모

작성: 2026-06-01 · 잠금: M2 1주차 빠른 시드 + M3 채널 확장 단위 분리

## 1. 패러다임 변화 한 줄

> **메타 먼저 → 글 먼저**. 학생이 가장 자연스럽게 갖고 오는 "글 그 자체"를 입력의 첫 행위로.
> 메타 입력은 글이 이미 화면에 있는 상태에서 — 시각적 무게감 ↓, 의사결정 부담 ↓.

기존: Block A(과제 메타 4~5칸) → Block B(글 textarea) → Block C(채점 CTA)
신규: **Step 1**(글 입력) → **Step 2**(글 미리보기 접힘 + 메타 입력) → **Step 3**(채점 진행)

## 2. 잠금된 Step 설계 (v1)

### Step 1 — "글을 넣어 주세요"
```
┌──────────────────────────────────────────────┐
│ ○○님, 어디서 가져온 글이든 받아드릴게요    │
│                                              │
│   [✏ 직접 입력/붙여넣기 — 큰 textarea]       │
│                                              │
│  📎 파일(DOCX·HWP·TXT)         [M3]          │
│  📷 사진(폰 카메라/갤러리)       [M3]         │
│  🔗 링크(Docs·Notion·블로그)     [M3]         │
│                                              │
│  [드래그앤드롭도 위 영역에서 작동]           │
└──────────────────────────────────────────────┘
```

- **출력**: 학생 글 텍스트 1건 (정규화 전)
- **시각 신호**: 글자수·진척 인디케이터 (#10/PR #23 재사용, Step 1 안에 배치)
- **다음 단계 활성 조건**: `char ≥ 50` (BODY_MIN)

### Step 2 — "내용 확인 + 과제 정보"
```
┌──────────────────────────────────────────────┐
│ ▸ 내 글 미리보기 (접힘, 클릭 시 펼침)         │
│   "재생 가능 에너지는… (3줄 요약)" / 646자     │
│   [수정] → Step 1으로                        │
│                                              │
│ ▸ 과제 정보 (직접 입력)                       │
│   학년     [고1 ▾]   ← /me 프로필 prefill   │
│   과목     [국어 ▾]   ← /me 프로필 prefill   │
│   장르     [감상문·독후감 ▾] ← prefill      │
│   목표 분량 [800] 자                         │
│   과제문   [textarea 50~1000자]              │
│                                              │
│ [채점받기]  (~12초 안내)                     │
└──────────────────────────────────────────────┘
```

- 미리보기는 기본 **접힘** — 화면 무게감↓, 펼치면 전체 텍스트
- /me 프로필 있으면 학년·과목·자주 쓰는 장르 prefill (현재 동작 유지)
- 실제 학생 입력 부담: 3~5칸 (프로필 prefill로 가변)

### Step 3 — "AI가 평가 중…"
- 진행 표시 (~12초 안내, 현 패턴 그대로)
- 완료 → 결과 화면 + `/results` 자동 저장 (#11/PR #29)

## 3. 입력 채널 4개 (M3 스코프)

| 채널 | M2 | M3 |
|---|---|---|
| 직접 입력/붙여넣기 | ✅ 항상 | (유지) |
| 클립보드 자동 감지 | ✅ 1주차 #B | (유지) |
| 드래그앤드롭 (textarea 자체) | ✅ 1주차 #C | (유지) |
| TXT 파일 업로드 | ✅ 1주차 #C | (유지) |
| DOCX 업로드 (mammoth) | ⏳ | ✅ |
| HWP 업로드 (hwp.js or Naver Clova) | ⏳ | ✅ |
| 사진 OCR (Naver Clova OCR) | ⏳ | ✅ |
| 링크 추출 (readability + cheerio + allowlist) | ⏳ | ✅ |

**AI 메타 추정은 제외** — 학생이 직접 입력. AI는 글만 본다.

## 5. v1 default 의사결정 (사용자 redirect 가능)

| 항목 | default | 근거 |
|---|---|---|
| /me 프로필 prefill 강도 | **학년·과목·자주 쓰는 장르 모두** (현재 동작 유지) | 사용자가 다시 입력하는 마찰 최소화. M3에서 LRU 5건 학습으로 강화. |
| DnD 영역 | **큰 textarea 자체에 떨어뜨림** (별도 dropzone X) | 사용자 sketch와 일치, UI 요소 추가 0. 사진/DOCX/HWP는 M3 별 channel picker. |
| TXT 인코딩 | **UTF-8 only v1**, EUC-KR은 후속 | 현 한국 학교 환경 대부분 UTF-8. EUC-KR 자동 감지는 후속 비용 대비 효과 적음. 실패 시 친절한 에러 카피. |
| 클립보드 권한 거절 | **조용히 폴백** (배너 자체 미노출) | 권한 거절·미지원 브라우저는 안 보임. 사용자 학습 부담 0. |

이 4개 default는 사용자 검토 후 PR에서 변경 가능 — `/try` 시연 결과 다른 선택이 더 자연스러우면 redirect.

## 6. 의존성 / 선행 조건 (M3 진입 시)

- **Vercel Hobby → 조직·Pro 이관** (memory `vercel-hobby-to-org-migration`)

  - 외부 API 비용 추적 (Clova OCR quota)
  - 함수 timeout 한계
- **법적 검토 — 미성년자 데이터 처리 정책 업데이트**

  - "파일/사진은 서버에 저장하지 않고 즉시 텍스트만 추출" 명시
  - EPO 재승인 필요

## 7. M3 채널 우선순위 (한국 학교 환경 추정)

추정 우선순위 (사용자 확정 필요):
1. **HWP** — 공교육 표준 포맷, 가장 큰 수요
2. **DOCX** — 사교육·대학 작문 비중
3. **사진 OCR** — 손글씨/인쇄물, 모바일 first
4. **링크** — Google Docs·Notion·블로그, IT 친화 학생

## 8. v0 → v1 전환 기준 (시연 후 평가)

- 학생 5명 dogfood — Step 1에서 글을 어디로 가져왔는지 logged (수동 메모, 자동 logging X)
- 시연 결과 "메타 먼저가 더 좋다" 의견 ≥3/5 → swap 롤백 검토
- "TXT 외 채널 필요" 의견 ≥2/5 → M3 우선순위 재정렬
